21. In your opinion, why does the average turnaround time and waiting time vary per algorithm


Because different CPU scheduling algorithms result in different completion times for the same set of processes. This is because the process timeout differs when we change the CPU scheduling algorithm.

CPU SCHEDULING ALGORITHM

CPU scheduling is a matter of determining which processes in the ready queue are allocated to the CPU. There are several CPU scheduling algorithms, including:

- First Come, First Served (FCFS) Scheduling Algorithm.

- Shortest Job First Scheduling Algorithm.

- Priority Scheduling Scheduling Algorithm (priority schedule).

- Round Robin Scheduling Algorithm.

Each algorithm is measured "turnaround time" and "waiting time" to compare performance with other algorithms. And to measure turnaround time and waiting time, the "Gant Chart" is used. CPU time (Burst Time) requires that all processes are assumed to be known. The arrival time for each process in the ready queue is assumed to be known.

First Come, First Served (FCFS) Scheduling Algorithm

The process that first requests time to use the CPU is served first. And the average waiting time (Average waiting time) is quite high. The FCFS scheduling algorithm is a non-Pr:eemptive scheduling strategy because once CPU is allocated to a process, that process will continue to use CPU until it releases it, i.e. if the process stops or requests I/O.

Shortest Job First Scheduler Algorithm

This algorithm is used when the CPU is fr:ee, the process that has the shortest time to complete gets priority. If two or more processes have the same time, the FCFS algorithm is used to solve the problem.

Priority Scheduling Algorithm (priority schedule)

SJF at (Shortest Job First) scheduling is a special case of the Priority scheduling algorithm. Priorities can be associated with each process and CPU is allocated to the process with the highest priority. For the same priority done with FCFS.  

Round Robin Scheduling Algorithm.

The Round Robin (RR) algorithm is designed for time sharing systems. This algorithm is simil4r to the FCFS scheduler, but pr:eemption is added for switches between processes. The ready queue is treated or considered as a circul4r queue.
